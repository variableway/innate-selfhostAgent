> 状态: ✅ 已完成 (有差异)

## 实现概览

在 `apps/desktop/` 下创建了 Tauri v2 + React 桌面应用，使用 Vite 作为构建工具。

## 技术栈

| 技术 | 版本 | 说明 |
|------|------|------|
| Tauri | v2.x | 桌面应用框架 |
| React | v19.x | UI 框架 |
| Vite | v7.x | 构建工具 |
| TypeScript | v5.8 | 类型系统 |
| Tailwind CSS | v3.4 | 样式框架 |

## 与原始需求的差异

| 原始需求 | 实际实现 | 说明 |
|----------|----------|------|
| Next.js 15 | React 19 + Vite | 使用 Vite 替代 Next.js，更适合桌面应用 |
| pnpm workspace | npm | 暂未配置 monorepo workspace |
| App Router | 不适用 | React + Vite 不使用 App Router |

## 为什么使用 React + Vite 而非 Next.js

1. **桌面应用特性**: Tauri 桌面应用不需要 SSR/SSG
2. **启动速度**: Vite 开发服务器启动更快
3. **配置简单**: 无需处理 Next.js 的复杂配置
4. **打包大小**: Vite 打包更轻量

## 运行方式

```bash
cd apps/desktop

# 安装依赖
npm install

# 开发模式
npm run tauri dev

# 构建
npm run tauri build
```

## 配置说明

### Tauri 配置 (`tauri.conf.json`)

```json
{
  "build": {
    "beforeDevCommand": "npm run dev",
    "devUrl": "http://localhost:1420",
    "beforeBuildCommand": "npm run build",
    "frontendDist": "../dist"
  },
  "app": {
    "windows": [
      {
        "title": "Executable Tutorial",
        "width": 1280,
        "height": 800
      }
    ]
  }
}
```

### Vite 配置 (`vite.config.ts`)

```typescript
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";

export default defineConfig({
  plugins: [react()],
  server: {
    port: 1420,
    strictPort: true,
  },
});
```

## 注意事项

1. **CSS 导入**: 确保 `main.tsx` 导入 CSS 文件
   ```typescript
   import "./index.css";
   ```

2. **Tailwind 版本**: 使用 v3 而非 v4（配置方式不同）

3. **开发端口**: 固定使用 1420 端口
