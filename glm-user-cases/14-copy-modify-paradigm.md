# Copy-Modify：AI 时代的"拿来主义"

## 一、核心理念

**不要从零开始。从别人的东西开始。**

```
传统模式:  需求 → 学习 → 设计 → 实现    （门槛高，周期长）
Copy-Modify: 看到好的 → 拿来 → 改成自己的  （门槛低，立刻用）
```

这个理念不新。Notion 模板、GitHub Template、Fork 仓库、小红书抄作业……本质都是 copy-modify。

**但 AI 让 copy-modify 变得更强了**：以前 copy 过来需要自己手动改，现在 AI 帮你改。

---

## 二、两类用户，两种深度

### 小白用户：Copy → 直接用

小白不需要"修改"。他只需要看到别人的东西好用，**一键抄过来**。

```
别人的减脂食谱 → 一键复制 → 变成我的食谱
别人的记账分类 → 一键复制 → 变成我的分类
别人的 XHS 分析模板 → 一键复制 → 变成我的模板
别人的每周习惯 → 一键复制 → 变成我的习惯
```

**小白的动作只有两个：看见 → 抄。不需要改。**

### 有基础用户：Copy → AI 帮改 → 变成自己的

有基础的用户不满足于"直接用"，他想**改成适合自己的**。

```
别人的减脂食谱 → "我不吃海鲜" → AI 自动调整
别人的记账分类 → "我加一个宠物类" → AI 自动扩展
别人的 Prompt → "改成日语教学" → AI 自动重写
别人的工作流 → "加一步自动备份" → AI 自动插入
```

**有基础的动作是三个：看见 → 抄 → 告诉 AI 怎么改。**

## 三、AI 场景下的 Copy-Modify 具体形态

### 3.1 Prompt 模板（最核心）

**这是 AI 时代 copy-modify 最直接的落地。**

小白看到的：
```
┌─────────────────────────────────────────┐
│ 📋 好用的 Prompt                        │
│                                         │
│ 🍜 减脂食谱生成器  ⭐ 2.3k 人用         │
│ "每天 1800 大卡减脂食谱..."              │
│                                         │
│ 💰 消费分析师     ⭐ 1.8k 人用         │
│ "分析本月消费，找出省钱点..."            │
│                                         │
│ 📱 XHS 内容提取器  ⭐ 1.5k 人用        │
│ "从小红书笔记中提取干货..."              │
│                                         │
│ 📖 读书笔记整理器  ⭐ 1.2k 人用        │
│ "把 PDF 变成 FlashCard..."              │
│                                         │
│        [一键使用]  [看看里面写了什么]     │
└─────────────────────────────────────────┘
```

**小白**：点"一键使用" → 直接开始对话
**有基础**：点"看看里面写了什么" → 看到 Prompt 原文 → 改几句话 → 保存为自己的

```typescript
// Prompt 模板数据结构
interface PromptTemplate {
  id: string;
  name: string;
  description: string;
  category: '饮食' | '财务' | '学习' | '效率' | '分析' | '创意';
  prompt: string;                 // 完整 Prompt
  variables: PromptVariable[];    // 可修改的变量
  author: string;
  useCount: number;               // 使用人数
  rating: number;
  tags: string[];
}

interface PromptVariable {
  name: string;                   // 如 "dailyCalories"
  label: string;                  // 如 "每日热量目标"
  type: 'number' | 'text' | 'select' | 'boolean';
  defaultValue: any;
  options?: string[];             // select 类型的选项
}

// 小白：填变量就行
// 有基础：直接改 Prompt 原文
```

**例子**：

```
减脂食谱生成器的 Prompt 模板：

---
你是专业营养师。根据以下要求生成一周食谱：
- 目标: {{goal}}
- 每日热量: {{dailyCalories}} kcal
- 人数: {{people}}
- 忌口: {{restrictions}}
- 偏好: {{preferences}}
---

小白看到的是：
  [目标: 减脂 ▾]  [热量: 1800]  [人数: 1 ▾]
  [忌口: _______]  [偏好: _______]
  [开始生成 →]

有基础看到的是：
  完整 Prompt 文本 → 直接编辑 → 保存为自己的版本
```

## 四、Copy-Modify 的完整闭环

```
发现 → 抄 → 用 → 改 → 分享

1. 发现: 浏览模板库 / 看到别人分享 / AI 推荐
2. 抄:   一键复制到自己的空间
3. 用:   直接用（小白）或填参数后用
4. 改:   告诉 AI 怎么改 → AI 帮你改
5. 分享: 改好了觉得不错 → 分享给其他人
         → 别人又可以抄你的
```

**这形成了一个飞轮**：用户既是消费者（抄别人的），也是生产者（改好后分享出去）。AI 是中间的加速器。

## 五、技术实现

### 5.1 模板库数据结构

```typescript
interface Template {
  id: string;
  type: 'prompt' | 'workflow' | 'lifestyle' | 'knowledge';
  name: string;
  description: string;
  author: {
    id: string;
    name: string;
  };
  content: string;               // 模板内容（Prompt 文本 / JSON 工作流 / 等）
  variables: TemplateVariable[];
  // 统计
  forkCount: number;              // 被抄了多少次
  useCount: number;               // 被用了多少次
  rating: number;
  // 来源追踪（copy-modify 链）
  forkedFrom?: string;            // 从哪个模板抄的
  tags: string[];
  createdAt: number;
}
```

### 5.2 Copy 动作

```typescript
// 一键复制
function forkTemplate(templateId: string): Template {
  const original = getTemplate(templateId);
  return {
    ...original,
    id: generateId(),
    author: { id: currentUser.id, name: currentUser.name },
    forkedFrom: templateId,
    forkCount: 0,
    useCount: 0,
    createdAt: Date.now(),
  };
}
```

### 5.3 Modify 动作（AI 辅助）

```typescript
// 用户描述想怎么改，AI 帮改
async function modifyWithAI(
  template: Template,
  instruction: string
): Promise<Template> {
  const modifiedContent = await glm.chat({
    model: 'GLM-5.1',
    messages: [{
      role: 'user',
      content: `
        这是原始模板:
        ${template.content}

        用户想这样改:
        ${instruction}

        请修改模板内容，保持结构和格式，只改用户要求的部分。
        输出修改后的完整模板。
      `
    }]
  });

  return { ...template, content: modifiedContent };
}
```

## 六、对两类用户的价值

### 小白用户

| 没有模板 | 有模板 |
|----------|--------|
| "AI 能帮我做什么？" | "这个食谱模板不错，我试试" |
| 面对空白对话框发呆 | 面对填空题直接填 |
| 不知道怎么描述需求 | 模板已经帮他想好了 |
| 用了 1 次就走了 | 每次都有新模板可以抄 |

**模板降低了 AI 的使用门槛，从"会提问"降级到"会抄作业"。**

### 有基础用户

| 没有模板 | 有模板 |
|----------|--------|
| 自己从头写 Prompt | 看别人的 Prompt，改几句就变成自己的 |
| 调试半天不理想 | 从已经验证的模板开始改 |
| 只能自己用 | 改好了可以分享，积累影响力 |
| 重复造轮子 | 站在别人肩膀上 |

**模板让有基础的用户从"创造者"变成"策展人"——不需要发明，只需要挑选 + 改进。**

## 七、和"随手记"的关系

Copy-Modify 是随手记的**内容层**，随手记是**交互层**。

```
随手记（交互层）: 一个输入框，用户只管发
  │
  ├─ 发的内容 → AI 自动分类
  │
  └─ 分类结果 → 匹配模板
     │
     ├─ 记账 → 匹配"消费分析"模板
     ├─ 食物 → 匹配"饮食记录"模板
     ├─ 心情 → 匹配"情绪日记"模板
     ├─ 链接 → 匹配"内容提取"模板
     └─ "帮我弄个食谱" → 弹出食谱模板库

模板是后台运转的。用户不需要知道模板的存在。
但当他想"定制"的时候，模板就浮出水面了。
```

## 八、优先做什么模板

| 模板 | 目标用户 | 为什么先做 |
|------|----------|-----------|
| 减脂食谱生成器 | 小白 | 高频 + 直观 |
| 消费月报分析 | 小白 | 和记账联动 |
| XHS 干货提取 | 小白 | 和 XHS 分析联动 |
| 读书笔记→FlashCard | 有基础 | 和下载书籍联动 |
| 自定义 Prompt 编辑器 | 有基础 | 让有基础用户创造模板给小白用 |

**先把前 4 个做成内置模板。第 5 个（编辑器）让有基础用户用起来后，模板库自然就丰富了。**
