## 第一部分：为什么因果关系重要

### 相关 ≠ 因果

经典例子：

- **观察**：冰淇淋销量增加时，溺水事故也增加
- **错误结论**：吃冰淇淋导致溺水？
- **真相**：夏天高温同时导致了两者

```
高温 ──→ 冰淇淋销量增加
  │
  └──→ 游泳人数增加 → 溺水事故增加
```

冰淇淋和溺水是**相关**的，但没有**因果关系**。真正的共同原因是"高温"。

### AI 的因果陷阱

现代 AI（包括大语言模型）主要基于**统计相关性**：

| 问题 | AI 的回答 | 真正原因 |
|------|----------|---------|
| 医院比家里更危险？ | 住院死亡率 > 居家死亡率 | 重病的人才去医院 |
| 公立学校不如私立？ | 私立升学率更高 | 家庭背景差异 |

如果 AI 只看相关性做决策，可能导致错误的结论和决策。

## 第二部分：因果推断基础

### 因果关系的三个层次

Judea Pearl（图灵奖得主）提出了**因果阶梯**：

```
第三层（反事实）："如果我当时没吃药，会怎样？"
    ↑
第二层（干预）："如果我吃了这个药，会怎样？"
    ↑
第一层（观察）："我看到了什么规律？"
```

**大多数 AI 目前只停留在第一层。**

### 因果图（DAG）

用有向无环图（DAG）表示因果关系：

```
      [高温]
       ↙   ↘
[冰淇淋]   [游泳]
              ↓
           [溺水]
```

箭头表示因果方向。通过分析因果图，我们可以：
- 找到真正的因果路径
- 识别混杂因素（同时影响两个变量的因素）
- 设计更好的干预策略

### do-算子 — "如果做了会怎样"

Pearl 引入了 `do(X)` 算子来表示干预：

- `P(Y|X)` — 观察到 X 时 Y 的概率（相关性）
- `P(Y|do(X))` — 人为设置 X 时 Y 的概率（因果性）

**例子**：
- `P(康复|吃药)` — 观察到吃药的人中康复的比例
- `P(康复|do(吃药))` — 让所有人吃药后康复的比例

这两个概率可能不同，因为吃药的人可能本身就更关注健康。

## 第三部分：优化方法

### 从梯度下降到高级优化

在基础教程中我们学了梯度下降。现在看看更高级的方法：

| 方法 | 说明 | 适用场景 |
|------|------|---------|
| SGD | 随机梯度下降 | 通用 |
| Momentum | 带动量的SGD | 加速收敛 |
| Adam | 自适应学习率 | 深度学习默认选择 |
| L-BFGS | 二阶优化 | 小数据集精确优化 |

### 正则化 — 防止过拟合

**过拟合**：AI 记住了训练数据的每个细节，但不能泛化到新数据。

```
好的模型：抓住趋势          过拟合：记住了每个噪声点
    ╱  ╲                      ╱╲╱╲╱╲╱╲
   ╱    ╲                    ╱        ╲
  ╱      ╲                  ╱          ╲
```

正则化方法：

| 方法 | 说明 | 类比 |
|------|------|------|
| L1 正则化 | 让部分参数变为零 | 做减法，去掉不重要的 |
| L2 正则化 | 让参数尽量小 | 让所有参数都"保守"一点 |
| Dropout | 随机丢弃部分神经元 | 团队不能依赖某一个人 |

## 第四部分：因果推断在 AI 中的应用

### 1. 可解释 AI（XAI）

因果推断帮助回答："AI 做出这个决策，**因为**什么？"

```
AI 拒绝了贷款申请
  → 因果分析：是因为收入不足？还是因为性别/种族（不公平）？
```

### 2. 推荐系统

因果推荐解决"偏差"问题：

- **观察数据**：用户只看了推荐的内容 → 偏向推荐算法的历史
- **因果方法**：如果推荐了不同的内容，用户会怎样？

### 3. 医疗 AI

```
传统 AI：症状A和疾病B经常一起出现
因果 AI：症状A是否导致了疾病B？还是疾病B导致了症状A？
```

## 关键概念速查

| 概念 | 一句话解释 | AI 中的意义 |
|------|-----------|-----------|
| 相关性 | 两件事一起出现 | 大多数 AI 只能发现这个 |
| 因果性 | 一件事导致另一件事 | 真正的智能需要理解这个 |
| 混杂因素 | 同时影响两件事的隐藏原因 | 可能导致错误结论 |
| 反事实 | "如果当时不同会怎样" | 最高层的因果推理 |
| 正则化 | 防止模型过度记忆 | 提高泛化能力 |
