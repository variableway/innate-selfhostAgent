# AI 助手进阶 — 自定义技能

> OpenClaw 的技能系统就像手机上的 App Store。你可以从市场安装现成技能，也可以自己开发。本教程教你如何管理和定制技能，让 AI 助手更贴合你的需求。

---

## 前置要求

- 已完成 OpenClaw 基础教程（体验、搜索、邮件）
- 熟悉基本的命令行操作
- 了解 JSON 格式

---

## 技能系统架构

OpenClaw 技能由以下部分组成：

```
技能 (Skill)
├── 元数据 (metadata)     — 名称、描述、版本、作者
├── 触发条件 (triggers)   — 什么时候激活这个技能
├── 工具定义 (tools)      — 技能能做什么
├── 提示模板 (prompts)    — AI 如何使用这个技能
└── 配置 (config)         — API Key、参数等
```

技能安装后，AI 会根据对话内容自动判断何时使用哪个技能。

## 步骤 3：技能配置最佳实践

### 保持 5-10 个高频技能

安装太多技能会降低 AI 的响应速度和准确性。推荐策略：

```
"底盘技能"（必装）:
├── tavily-search    — 搜索是一切的基础
├── agent-browser    — 浏览器控制
├── gog             — 邮件
└── knowledge-base  — 知识库

"场景技能"（按需安装）:
├── 开发场景: github, code-reviewer, git-ops
├── 写作场景: blogburst, x-api
└── 运维场景: devops, aws-infra
```

### 技能优先级配置

在 `~/.claw/skills.json` 中设置技能优先级：

```json
{
  "priority": ["tavily-search", "github", "gog"],
  "disabled": ["deprecated-skill"]
}
```

## 步骤 5：自定义技能开发（进阶）

### 技能文件结构

```
my-skill/
├── skill.json       — 技能定义
├── prompts/
│   └── system.md    — 系统提示
└── tools/
    └── main.py      — 工具实现
```

### skill.json 示例

```json
{
  "name": "my-custom-skill",
  "version": "1.0.0",
  "description": "我的自定义技能",
  "triggers": ["自定义关键词"],
  "tools": [
    {
      "name": "my_function",
      "description": "执行自定义操作",
      "parameters": {
        "input": { "type": "string", "description": "输入参数" }
      }
    }
  ]
}
```

### 安装自定义技能

<RunnableCodeBlock code="claw skill install --local ./my-skill" language="sh" />
