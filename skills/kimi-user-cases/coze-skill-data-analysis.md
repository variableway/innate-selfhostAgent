# Coze Skill/Bot 数据分析方法论

> 分析日期：2026-04-16
> 适用对象：Coze 开发者、Bot 运营者、AI 产品经理

---

## 核心命题

**Coze 是国内最成熟的 AI Agent 平台，但它的数据分析能力却比较薄弱。**

官方只提供了基础的"对话次数"、"活跃用户数"等面板，但缺乏：
- 用户到底在问什么？
- 哪些节点在工作流里卡住了？
- 为什么用户用了一次就不再回来？
- 哪个插件最受欢迎？
- 同一类 Bot 的竞争力如何？

**本方法论的目标是：用"官方数据 + 第三方抓取 + 自建埋点"的组合拳，建立起一套完整的数据分析体系。**

---

## 第一部分：平台宏观生态分析

### 分析目标
了解 Coze 平台上哪些类型的 Bot/Skill 最受欢迎，市场空白在哪里，竞争格局如何。

### 数据来源

#### 1. 官方公开数据
- **Coze 商店 (coze.cn/store)**：每个 Bot 的"使用人数"、"点赞数"、"评分"
- **插件商店 (coze.cn/store/plugin)**：插件的"使用次数"、"评分"
- **工作流模板库**：模板的"使用次数"

#### 2. 第三方聚合数据
- **爬虫抓取**：用 Python + Selenium/Playwright 抓取商店页面的 Bot 列表
- **API 监控**：部分数据可以通过 Coze 的公开接口获取（需要逆向或抓包）

### 核心指标

| 指标 | 定义 | 获取方式 |
|------|------|---------|
| **Bot 总数** | 商店中公开上架的 Bot 数量 | 爬虫抓取 |
| **分类分布** | 各品类（工具/娱乐/学习/办公）的 Bot 占比 | 爬虫分类统计 |
| **头部集中度** | Top 100 Bot 占总使用量的比例 | 使用人数排序分析 |
| **新品存活率** | 上架 30 天后的 Bot 是否仍在更新 | 时间序列追踪 |
| **插件渗透率** | 使用某个插件的 Bot 数量 / 总 Bot 数量 | 插件页数据关联 |

### 分析维度

#### 维度 1：品类热度矩阵

```
        高使用人数
              │
    ┌─────────┼─────────┐
    │  明星品类 │  红海品类 │
    │  (蓝海)  │  (竞争激烈)│
低 Bot 数量 ───┼─────────┤ 高 Bot 数量
    │  冷门品类 │  潜力品类 │
    │  (观望)  │  (待验证) │
    └─────────┼─────────┘
              │
        低使用人数
```

**实操示例**：
- 抓取 5000 个 Bot 的数据
- X 轴：该分类下的 Bot 数量
- Y 轴：该分类下所有 Bot 的平均使用人数
- 找出"Bot 少但人均使用高"的明星品类（如"财务分析"可能只有 50 个 Bot，但平均使用人数是娱乐 Bot 的 3 倍）

#### 维度 2：头部 Bot 拆解

选取每个分类 Top 10 的 Bot，拆解以下维度：
- **Prompt 结构**：系统提示词的字数、角色设定复杂度
- **工作流节点数**：简单 Bot（< 5 个节点）vs 复杂 Bot（> 20 个节点）
- **插件依赖**：使用了哪些官方插件/自定义插件
- **更新频率**：最后一次更新是什么时候
- **发布渠道**：是否同时发布了飞书/微信/抖音版

**输出形式**：
```markdown
# Coze 头部 Bot 拆解报告

## 工具类 Top 10
| 排名 | Bot 名称 | 使用人数 | 工作流节点 | 核心插件 | 更新频率 | 渠道分布 |
|-----|---------|---------|-----------|---------|---------|---------|
| 1   | AI 写作助手 | 1.2M | 12 | 文本生成/搜索 | 每周 | 全渠道 |
| 2   | 会议纪要整理 | 890K | 8 | 语音转写/文档 | 每月 | 飞书+微信 |

## 关键洞察
1. 工具类头部 Bot 的节点数集中在 8-15 个（复杂度适中）
2. 90% 的头部 Bot 使用了"网页搜索"插件
3. 更新频率和使用人数呈强正相关（r=0.72）
```

#### 维度 3：插件生态分析

**分析插件市场的供需关系**：
- 哪些插件被大量 Bot 使用但供给不足？（机会）
- 哪些插件已经有大量替代方案？（红海）
- 哪些插件的使用人数在快速增长？（趋势）

**示例输出**：
```markdown
# Coze 插件市场分析

## 供不应求的插件类型
- 财务数据分析插件：只有 3 个，但被 1200+ Bot 引用
- 小红书内容生成插件：只有 2 个官方插件，质量一般

## 红海插件类型
- 天气查询插件：12 个，功能雷同
- 翻译插件：18 个，差异化很小

## 增长最快的插件（近 30 天）
- DeepSeek-R1 推理插件：+340%
- 飞书多维表格插件：+120%
- 抖音热点搜索插件：+98%
```

## 第二部分：单个 Bot 的运营数据分析

### 官方数据面板能看什么？

Coze 后台提供的"数据分析"模块包含：
- **总对话数**：累计对话轮数
- **活跃用户**：一定周期内发起过对话的用户数
- **新增用户**：新发起的用户
- **消息数**：用户发送的消息总数
- **渠道分布**：飞书 / 微信 / 抖音等各渠道的占比

**但这些数据非常粗粒度**，无法满足精细化运营需求。

### 补充数据来源 1：自建埋点（最关键）

#### 在 Prompt / 工作流中插入"数据记录节点"

Coze 支持调用外部 API，我们可以在工作流的关键节点后插入一个"数据上报"节点：

```
用户输入
    │
    ▼
[意图识别节点]
    │
    ├── 意图 A → [处理 A] → [数据上报：意图=A] → [输出 A]
    ├── 意图 B → [处理 B] → [数据上报：意图=B] → [输出 B]
    └── 意图 C → [处理 C] → [数据上报：意图=C] → [输出 C]
```

**数据上报节点的实现**：
调用一个你自己的 HTTP API（可以用腾讯云函数/阿里云函数/vercel），把以下数据发过去：

```json
{
  "bot_id": "你的 Bot ID",
  "user_id_hash": "用户 ID 的哈希值（脱敏）",
  "timestamp": "2026-04-16T09:23:00+08:00",
  "intent": "账单分析",
  "input_length": 45,
  "output_length": 320,
  "node_path": ["输入", "意图识别", "CSV解析", "数据上报", "输出"],
  "execution_time_ms": 2340,
  "is_error": false,
  "channel": "feishu"
}
```

**通过这种方式，你可以获得官方不提供的数据**：
- 用户最常问的 10 个意图是什么？
- 哪个工作流节点的执行时间最长？
- 哪个节点的报错率最高？
- 用户输入的平均长度和输出长度的比例？

### 补充数据来源 2：对话日志分析

Coze 后台支持查看"对话日志"（测试版本可能有限制），但如果是发布到你自己的飞书群/公众号，你可以：

#### 飞书群 Bot 的对话日志获取

如果你把 Coze Bot 发布到飞书群：
- 飞书开放平台可以获取群聊天记录
- 筛选出 @Bot 的消息和 Bot 的回复
- 用 LLM 批量分析对话主题、情绪、满意度

**Python 分析脚本示例**：
```python
import pandas as pd
from collections import Counter

# 假设你已经从飞书导出了对话数据
df = pd.read_csv("feishu_bot_conversations.csv")

# 1. 用户高频意图统计
intents = df["user_message"].apply(extract_intent_with_llm)
print(Counter(intents).most_common(10))

# 2. 对话轮数分布
session_lengths = df.groupby("session_id").size()
print(f"平均对话轮数: {session_lengths.mean():.1f}")
print(f"中位数对话轮数: {session_lengths.median():.1f}")

# 3. 用户留存分析
first_use = df.groupby("user_id")["timestamp"].min()
second_use = df.groupby("user_id")["timestamp"].nth(1)
retention = (second_use - first_use).dropna()
print(f"次日留存率: {(retention < pd.Timedelta(days=1)).mean():.1%}")
print(f"7日留存率: {(retention < pd.Timedelta(days=7)).mean():.1%}")

# 4. 时段分布
df["hour"] = pd.to_datetime(df["timestamp"]).dt.hour
print(df["hour"].value_counts().sort_index())
```

### 补充数据来源 3：A/B 测试数据

Coze 允许你创建多个版本的 Bot 进行测试。你可以：

1. **创建版本 A 和版本 B**
   - A 版：系统提示词强调"简洁输出"
   - B 版：系统提示词强调"详细输出"

2. **各分流 50% 的流量**（通过不同渠道或不同群发布）

3. **对比指标**：
   - 平均对话轮数（哪版用户更愿意多聊？）
   - 次日回访率（哪版用户更愿意回来？）
   - 平均消息长度（哪版用户输入更具体？）
   - 投诉/负面反馈率

## 第三部分：竞品 Bot 分析

### 分析目标
找到对标 Bot，学习其优点，发现其盲区。

### 数据来源

#### 1. 公开信息抓取
- Coze 商店页面的：名称、简介、使用人数、点赞数、更新时间
- Bot 的对话入口：可以实际体验和测试（输入相同的问题，对比输出）

#### 2. "神秘顾客"测试法
对目标竞品 Bot 发送同一套测试问题，记录输出：

```markdown
# 竞品测试问题集（以"会议纪要整理"为例）

Q1: 基础能力
"请帮我整理以下会议纪要：
 参会人：张三、李四

观察维度：
- 是否识别出参会人？
- 是否提取了关键结论？
```

## 第四部分：用户行为深度分析

### 漏斗分析：从"看到"到"持续使用"

```
Step 1: 曝光
  ├── 指标：商店浏览量、搜索曝光量、社群分享次数
  └── 分析：用户是通过什么渠道发现这个 Bot 的？
  
Step 2: 首次对话
  ├── 指标：首次激活率（看到 Bot 后发起对话的比例）
  └── 分析：Bot 的名称、简介、欢迎语是否足够吸引人？
  
Step 3: 完成首次任务
  ├── 指标：首次任务完成率（对话后得到满意结果的比例）
  └── 分析：首次对话是否让用户获得了"Aha Moment"？
  
Step 4: 次日回访
  ├── 指标：次日留存率
  └── 分析：用户是否有理由第二天再回来？
  
Step 5: 持续使用（每周 3 次以上）
  ├── 指标：周活跃率、月活跃率
  └── 分析：Bot 是否融入了用户的日常工作流？
  
Step 6: 推荐传播
  ├── 指标：分享率、邀请率、自然增长占比
  └── 分析：用户是否愿意把这个 Bot 推荐给朋友？
```

### 关键行为指标（北极星指标建议）

**对于工具类 Bot**：
> 北极星指标 = **周活跃用户数 × 人均任务完成数**
> （不仅要人来，还要把事情做成）

**对于娱乐类 Bot**：
> 北极星指标 = **人均对话轮数 × 次日留存率**
> （粘性比总量更重要）

**对于内容生产类 Bot**：
> 北极星指标 = **人均内容导出/分享次数**
> （用户真正用产出的内容去做事了，才算有价值）

## 第五部分：变现与商业化分析

### Coze 目前的变现路径

| 路径 | 说明 | 数据关注点 |
|------|------|-----------|
| **免费引流** | 用免费 Bot 获取用户 | CAC（获客成本）、渠道效率 |
| **付费 Bot/服务** | Coze 支持部分商业化功能 | 付费转化率、ARPU、LTV |
| **私域转化** | 把 Bot 用户导到微信/社群/课程 | 导粉率、私域转化率 |
| **企业服务** | 为企业定制 Bot 和工作流 | 客单价、交付周期、复购率 |

### 变现效率分析框架

```
收入 = 流量 × 转化率 × 客单价 × 复购率

流量层：
  - 商店自然流量占比
  - 社交媒体引流占比
  - 老用户推荐占比

转化层：
  - 免费→付费的转化率
  - 试用→订阅的转化率
  - 单次→复购的转化率

价格层：
  - 不同定价下的转化率弹性
  - 会员权益的感知价值

留存层：
  - 月流失率
  - 用户生命周期价值（LTV）
  - LTV / CAC 比值（健康值应 > 3）
```

## 第六部分：可落地的分析工具箱

### 工具 1：Coze Bot 商店爬虫

```python
# 用 Playwright 抓取 Coze 商店数据
from playwright.sync_api import sync_playwright
import json

def scrape_coze_store(category="all", pages=10):
    results = []
    with sync_playwright() as p:
        browser = p.chromium.launch()
        page = browser.new_page()
        
        for i in range(1, pages + 1):
            url = f"https://www.coze.cn/store/agent?page={i}&category={category}"
            page.goto(url)
            page.wait_for_selector(".agent-card")
            
            cards = page.query_selector_all(".agent-card")
            for card in cards:
                results.append({
                    "name": card.query_selector(".title").inner_text(),
                    "users": card.query_selector(".user-count").inner_text(),
                    "likes": card.query_selector(".like-count").inner_text(),
                    "category": category
                })
        
        browser.close()
    
    with open("coze_bots.json", "w") as f:
        json.dump(results, f, ensure_ascii=False)
    
    return results
```

### 工具 3：对话意图自动分类

```python
import openai

def classify_intent(user_message):
    prompt = f"""
    请将以下用户消息分类到一个意图标签中。
    可选标签：账单分析、摄影点评、行程规划、情绪倾诉、技术咨询、闲聊、投诉反馈、其他
    
    用户消息：{user_message}
    
    只输出标签名称，不要解释。
    """
    
    response = openai.ChatCompletion.create(
        model="gpt-3.5-turbo",
        messages=[{"role": "user", "content": prompt}]
    )
    
    return response.choices[0].message.content.strip()
```

## 第七部分：给不同类型开发者的分析建议

### 如果你是个人开发者（做 1-2 个 Bot）

**优先级**：
1. **自建埋点**（最重要）：在关键节点加 HTTP 上报
2. **对话日志分析**：每周导出一次对话记录，用 LLM 批量分类意图
3. **竞品测试**：每月找 3 个同类 Top Bot 做对比测试

**最小可行分析栈**：
- 数据上报：Vercel 免费云函数
- 数据存储：飞书多维表格（免费）
- 可视化：飞书多维表格自带图表
- 成本：¥0

### 如果你是小型团队（做 5-10 个 Bot）

**优先级**：
1. 搭建统一的**数据仓库**（PostgreSQL / ClickHouse）
2. 建立**标准化的事件上报 schema**
3. 用 **Metabase / Grafana** 做可视化看板
4. 定期（每周）输出**Bot 健康度报告**

**推荐技术栈**：
- 数据上报：Vercel / 阿里云函数
- 数据仓库：TiDB Cloud 或 AWS RDS
- 可视化：Metabase（开源免费）
- 自动化：n8n 定时生成周报并推送到飞书群

### 如果你是 MCN / 培训机构（批量做 Bot）

**优先级**：
1. 建立**Bot 商店监控体系**：抓取全平台数据，发现趋势和机会
2. 建立**标准化生产流水线**：从需求 → Bot 搭建 → 上架 → 数据监控
3. 用**数据驱动选题**：什么类型的 Bot 在增长，就批量做什么

**关键指标**：
- 单个 Bot 的 ROI（开发成本 vs 带来的流量/收入）
- 爆款率（Top 1000 中你的 Bot 占比）
- 矩阵协同效应（A Bot 的用户是否会被引导到 B Bot）

## 一句话总结

> **Coze 官方的数据面板只能告诉你"有多少人用了"，但无法告诉你"他们为什么来、为什么走、哪里卡住了"。真正的数据分析需要自建埋点 + 对话日志挖掘 + 竞品测试的三重组合拳。对个人开发者来说，最小成本的分析栈是"Vercel 云函数 + 飞书多维表格 + Python 脚本"；对团队来说，应该尽早建立统一的数据仓库和标准化事件 schema。数据不是为了好看，而是为了让你知道：下一个版本该优化什么。**
