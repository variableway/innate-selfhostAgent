# WayToAGI 批量转化为教程 — Pipeline 调研报告

## 1. WayToAGI 内容概述

**WayToAGI（通往AGI之路）** 是国内最大的中文 AI 开源知识库，托管在飞书 Wiki 上。

| 指标 | 数据 |
|------|------|
| 学习者覆盖 | ~900万人，37个城市 |
| 文章数量 | 10,000+ 篇 |
| 创建时间 | 2023年4月至今 |
| 托管平台 | 飞书 Wiki (waytoagi.feishu.cn) |
| 已有静态站 | wiki.waytoagi.com（使用 feishu-pages 导出） |

### 内容分类

| 分类 | 描述 | 教程转化价值 |
|------|------|-------------|
| 初学者路径 | 基础 AI/AIGC 概念，通俗易懂 | ⭐⭐⭐⭐⭐ 直接转化为 beginner 教程 |
| 开发者路径 | AI 应用构建、API 集成、模型部署 | ⭐⭐⭐⭐⭐ 转化为 intermediate 教程 |
| 模型用户路径 | 大语言模型和 AI 工具使用指南 | ⭐⭐⭐⭐ 需要拆分和重组 |
| 提示工程 | 精选 AI 提示词集合 | ⭐⭐⭐⭐ 可独立成教程系列 |
| AIGC 实战 | 文本/图像/音频/视频 AI 生成 | ⭐⭐⭐⭐⭐ 高价值实战内容 |
| AI 工具目录 | 分类 AI 网站和工具 | ⭐⭐⭐ 工具索引，需重新组织 |
| 行业新闻 | 每日更新 | ⭐⭐ 不适合转化 |
| 活动与竞赛 | 线下聚会、竞赛信息 | ⭐ 不适合转化 |

## 2. 转化 Pipeline 设计

### Pipeline 总览

```
[飞书Wiki内容] → Step1:提取 → Step2:解析 → Step3:分类 → Step4:转化 → Step5:质量检查 → Step6:发布
```

### Step 1: 内容提取

**目标**: 从飞书 Wiki 批量导出所有文档为 Markdown

**推荐方案: feishu-pages**（已被 WayToAGI 团队验证）

| 工具 | 可行性 | 复杂度 | 需要授权 | MDX输出质量 |
|------|--------|--------|---------|------------|
| **feishu-pages** (推荐) | ⭐⭐⭐⭐⭐ | 低-中 | 是 | 优秀 |
| feishu2md (Go CLI) | ⭐⭐⭐⭐ | 低 | 是 | 良好 |
| feishu_batch_download (Python) | ⭐⭐⭐⭐ | 低-中 | 是 | 良好 |
| 飞书 API 直接调用 | ⭐⭐⭐⭐ | 中 | 是 | 良好 |
| Web 抓取 | ⭐⭐ | 高 | 否 | 差 |
| 复用 wiki.waytoagi.com 导出 | ⭐⭐⭐ | 中 | 否 | 中 |

**feishu-pages 详细方案**:
- 仓库: https://github.com/longbridge/feishu-pages
- 安装: `yarn add feishu-pages`
- 功能: 保留目录树结构、输出 Markdown + frontmatter
- 自动化: 支持 GitHub Actions CI/CD
- **WayToAGI 已用它构建 wiki.waytoagi.com，验证了可行性**

**前置条件**:
- 飞书开放平台注册应用 (App ID + App Secret)
- Wiki 空间所有者授权 `wiki:wiki:readonly` + `docs:doc:readonly` 权限
- 或使用用户访问令牌（页面已公开共享即可）

**替代方案: 直接复用 wiki.waytoagi.com**:
- 已有静态导出，可直接抓取 Markdown 内容
- 无需授权，但需要归属标注
- 可能不是最新版本

### Step 2: 内容解析与结构化

**目标**: 将提取的 Markdown 解析为结构化数据

**工具与技术**:
- **unified/remark** 生态系统: Markdown → AST → 结构化数据
- **gray-matter**: 解析 frontmatter 元数据
- **自研解析器**: 识别教程特有的结构（章节、代码块、提示框）

**可行性**: ⭐⭐⭐⭐⭐
- Markdown 解析技术成熟
- 飞书导出的 Markdown 结构相对统一
- 可使用 AI 辅助识别内容结构

### Step 3: 内容分类与分级

**目标**: 按难度和主题自动分类内容

**方法**:
- **规则分类**: 基于目录结构和标题关键词
  - "入门"/"基础"/"介绍" → beginner
  - "进阶"/"实战"/"高级" → intermediate
- **AI 辅助分类**: 使用 LLM 分析内容难度
  - 输入: 文章内容
  - 输出: {difficulty, category, tags, prerequisites}
- **人工审核**: AI 分类结果 + 人工校对

**可行性**: ⭐⭐⭐⭐
- 规则分类可靠，覆盖 60-70% 场景
- AI 分类可提升到 85-90% 准确率
- 需要少量人工校对

### Step 4: 内容转化（Markdown → MDX 教程）

**目标**: 将通用 Markdown 转化为项目 MDX 教程格式

**转化规则**:
1. 添加标准 frontmatter（title, description, difficulty, duration, category, tags）
2. 代码块 → `<RunnableCodeBlock>` 组件
3. 平台相关内容 → `<PlatformTabs>` 组件
4. 添加学习目标和总结
5. 插入互动元素（验证步骤、练习）

**AI 辅助转化**:
- 使用 LLM 批量处理转化
- Prompt 模板化，确保输出格式一致
- 人工审核关键教程

**可行性**: ⭐⭐⭐⭐
- 格式转化技术可行
- AI 可处理 80% 的转化工作
- 需要人工审核确保质量

### Step 5: 质量检查

**目标**: 确保转化后的教程质量和一致性

**检查项**:
- frontmatter 完整性校验
- 代码块语法正确性
- 链接有效性
- 内容连贯性
- 难度分级准确性

**工具**: 自研脚本 + AI 审核

**可行性**: ⭐⭐⭐⭐⭐

### Step 6: 发布

**目标**: 将教程集成到项目教程系统

**流程**:
1. MDX 文件放入 `public/skills/` 目录
2. 教程系统自动扫描和索引
3. 用户在应用中浏览和学习

**可行性**: ⭐⭐⭐⭐⭐

## 3. 商业化可行性分析

### 3.1 市场需求

| 维度 | 分析 |
|------|------|
| 目标用户 | 中文 AI 学习者（900万+），增长迅速 |
| 痛点 | 飞书 Wiki 内容零散、缺乏实践互动、无分级学习路径 |
| 差异化 | 交互式教程 + 可执行代码 + 分级学习 + 进度追踪 |
| 竞品 | 极客时间、Coursera、飞书原始内容 |

### 3.2 收入模型

| 模式 | 预估收入 | 可行性 |
|------|---------|--------|
| 免费增值 | 基础教程免费，高级教程付费 | ⭐⭐⭐⭐ |
| 订阅制 | 月/年费解锁全部教程 | ⭐⭐⭐⭐ |
| 企业授权 | 定制化教程内容给企业客户 | ⭐⭐⭐ |
| 广告/合作 | AI 工具推荐佣金 | ⭐⭐⭐ |

### 3.3 成本结构

| 成本项 | 估算 |
|--------|------|
| 内容提取 | 低（工具自动化） |
| AI 转化成本 | 中（LLM API 调用费） |
| 人工审核 | 中-高（质量保证） |
| 持续维护 | 低（自动化更新 Pipeline） |

### 3.4 关键风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 飞书 API 权限获取 | 高 | 复用已有导出；联系 WayToAGI 合作 |
| 内容版权 | 高 | 标注来源、CC 协议、合作授权 |
| 内容质量不一致 | 中 | 多轮质量检查 + AI 审核 |
| 飞书格式变更 | 低 | 多种提取方案备份 |
| 市场竞争 | 中 | 差异化（交互式 + 本地执行） |

### 3.5 推荐策略

1. **Phase 1 (1-2周)**: 使用 feishu-pages 或复用 wiki.waytoagi.com 提取内容，手动转化 10-20 篇高质量教程作为种子内容
2. **Phase 2 (2-4周)**: 搭建自动化 Pipeline，AI 批量转化 + 人工审核
3. **Phase 3 (1-2月)**: 与 WayToAGI 社区建立合作，获得正式授权和持续内容更新
4. **Phase 4 (持续)**: 商业化探索，订阅/增值服务

### 3.6 结论

**商业化可行性: 中-高**

- **优势**: 目标用户群大、内容转化技术可行、差异化明确
- **挑战**: 内容授权是关键瓶颈，需要 WayToAGI 合作
- **建议**: 先以开源方式验证 Pipeline 和内容质量，再探索商业化
